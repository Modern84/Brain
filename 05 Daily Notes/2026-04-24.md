## Lokale LLMs Revert

Heute früh gestartet als konkretes Ziel. Nach Tests zurückgenommen.

### Was getestet wurde

**llama3.3:70b** (42 GB, Q4)
- Erster Versuch 09:xx: Crash → damals als Moonlock-OOM interpretiert. War's auch teilweise.
- Zweiter Versuch nach Cleanup: `500 Internal Server Error: model runner has unexpectedly stopped`.
- Rechnung: 42 GB Weights + KV-Cache + Activations ≈ 48–52 GB. System-Baseline nach Cleanup 24 GB (Chrome, Obsidian, Claude Desktop, Claude Code, Terminal, Finder, Kernel). **72 GB Bedarf auf 64 GB RAM** → OOM, unabhängig von Moonlock.

**qwen2.5:32b** (20 GB)
- Läuft technisch. System-Last: PhysMem 62G used / 882M unused. Funktioniert, aber mit 882 MB Puffer — Fusion parallel = nicht möglich.
- Qualität an Maschinenbau-Testprompt (Lagerschalenhalter vs. Lagerbock): **sachlich schwach**. Flüssiger Text, aber Inhalt teilweise erfunden. Kein Ersatz für Cloud-Claude bei fachlichen WEC-Fragen.

### Entscheidung

**Deinstallation aller lokalen LLMs.** Gründe:

1. **Permanenter RAM-Reserve-Kostet** (~20 GB für "vielleicht mal offline") steht in keinem Verhältnis zum Nutzen.
2. **Qualität unter Cloud-Claude** — für fachliche WEC-Arbeit (Bauteilbeschreibungen, technische Texte, Kundenanfragen) nicht produktionstauglich.
3. **Online-Verfügbarkeit praktisch gegeben** — Handy-Hotspot jederzeit verfügbar, reale Offline-Zeiten selten und kurz überbrückbar.
4. **WEC-sensitive-Daten-Argument bleibt gültig** — aber erst relevant ab Mac Studio Ultra 256GB (Juli). Bis dahin kein Usecase den Cloud-Claude nicht besser erfüllt.

### Cleanup

```bash
brew services stop ollama
ollama rm qwen2.5:32b llama3.3:70b llama3.2:3b
brew uninstall ollama
rm -rf ~/.ollama
# Aliase aus ~/.zshrc entfernen (q-on, q-off)
```

Rückgewinn: ~64 GB Disk (42 + 20 + 2), Service + Binary + Config weg.

### Lesson

**Hypothese geprüft, verworfen, weiter.** Nicht jede Idee die morgens überzeugt, überlebt den Mittag. Das ist kein Rückschritt, sondern sauberes Validieren. Die ursprüngliche Roadmap (Juli/August auf Mac Studio Ultra) steht — aber nicht als Zwischenschritt auf dem MacBook.

### Für die Zukunft

Wenn Mac Studio Ultra 256GB kommt, lokale LLMs neu evaluieren. Dann sind die Randbedingungen andere (50 GB für Modell von 256 GB = 20 % statt 75 %, und dedicated Desktop statt mobiler Daily-Driver).
